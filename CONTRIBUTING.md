# Contributing

Found a gap in the framework? Better security pattern? Improved UX component? We'd love your input.

## How to Contribute

1. **Fork** the repo
2. **Make your changes** — keep commits focused on one thing
3. **Open a PR** — explain what you changed and why

## What We're Looking For

- Security improvements (new attack vectors, better defenses)
- Additional integration patterns (new auth libraries, databases, frameworks)
- Country-specific compliance guidance
- Better frontend components
- Bug fixes in code examples
- Documentation clarity improvements

## Guidelines

- This is a reference framework, not an npm package. Code should be copy-paste ready.
- Every code snippet should include comments explaining the *why*, not just the *what*.
- No secrets, API keys, or real credentials in any file.
- Keep the docs practical. Show the pattern, explain the trade-off, move on.

## Questions?

Open an issue. We're friendly.
